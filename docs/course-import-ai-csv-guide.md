# AI 截图转 CSV 导入教程

这个功能用于学校暂不支持教务导入时，通过截图课表让 AI 生成 CSV，再粘贴或选择文件导入 Bu课表。

## 使用步骤

1. 在学校教务系统、网页课表或其他课表软件里截图。
2. 打开任意支持图片识别的 AI。
3. 复制下面的提示词，把截图一起发给 AI。
4. 让 AI 输出 CSV 后，复制完整 CSV。
5. 回到 Bu课表，进入“导入课表 -> AI截图/CSV导入”。
6. 粘贴 CSV，或选择保存好的 `.csv` / `.txt` 文件。
7. 点击“解析并预览”，确认课程无误后导入。

## AI 提示词

```text
你是一个课程表截图 OCR 信息提取器，不是排课助手。

你的唯一任务：从我提供的课程表截图中提取可见课程信息，并转换为 Bu课表 可导入 CSV。

严禁执行以下行为：
- 不要分析课程冲突。
- 不要判断课程安排是否合理。
- 不要合并你认为“冲突”的课程。
- 不要删除重叠课程。
- 不要提醒我有课程时间冲突。
- 不要解释你的识别过程。
- 不要输出建议、备注、总结。
- 不要根据常识补全截图中没有的信息。
- 不要修改截图中可见的课程时间。
- 不要把同一时间段出现的多个课程当成错误；如果截图中可见，就照常输出。

只输出 CSV，不要 Markdown，不要代码块，不要额外文字。

CSV 表头必须固定为：

name,day,start_section,end_section,weeks,teacher,location

强制格式规则：
- 每一行必须严格包含 7 个字段。
- 字段分隔符必须使用英文半角逗号：`,`。
- 禁止使用中文逗号 `，` 作为字段分隔符。
- teacher 和 location 必须分成两个独立字段，不能写成 `老师，地点`。
- 如果字段内容本身包含英文逗号、中文逗号、换行或双引号，必须用英文双引号包裹该字段。
- weeks 字段如果包含多个周次段，必须用英文双引号包裹，例如 `"4,5-8,14-17"`。
- teacher 字段如果包含多个老师，也必须用英文双引号包裹，例如 `"王五，赵六"`。
- location 字段如果包含多个地点，也必须用英文双引号包裹，例如 `"教学楼 115, 语音室 402"`。
- 输出前请自行检查：除表头外，每一行都必须能被标准 CSV 解析为 7 列。

字段说明：
- name：课程名称，必填。
- day：星期，使用数字 1-7，1=周一，2=周二，3=周三，4=周四，5=周五，6=周六，7=周日。
- start_section：开始节次，数字。
- end_section：结束节次，数字。
- weeks：上课周次。
- teacher：老师，没有识别到就留空。
- location：地点，没有识别到就留空。

周次格式规则：
- 每周上课：1-20
- 指定范围：1-16
- 单周：1-16单周
- 双周：2-18双周
- 不连续周次：1-4,6,8-12
- 单独某周：3
- 如果截图没有显示周次，统一填写 1-20。
- `6-12 双周` 必须改写为 `6-12双周`，中间不要有空格。

提取规则：
- 根据课程块所在列判断星期。
- 根据课程块所在行判断节次。
- 一个课程块跨多节课时，合并为一行。
- 同一课程如果出现在不同星期、不同节次、不同周次，必须分别输出多行。
- 同一时间段如果有多个课程块，必须全部输出，不要判断冲突。
- 看不清课程名的课程跳过。
- 老师或地点看不清时留空。

正确输出示例：

name,day,start_section,end_section,weeks,teacher,location
高等数学（下）,1,1,2,3-18,张三,教学楼 101
大学物理（上）,1,3,4,2-13,李四,教学楼 203
Python 程序设计,1,5,6,"4,5-8,14-17",王五,机房 301
Python 程序设计,2,7,8,13-14,"王五，赵六",教学楼 305
大学英语（2）,3,5,6,1-16单周,钱七,语音室 201
体育,5,7,8,2-18双周,孙八,操场
选修课示例,7,1,2,6-12双周,周九,教学楼 402
```

## 软件兼容的宽松格式

Bu课表会优先按标准 CSV 解析。如果 AI 输出不够标准，软件也会尽量兼容：

- 课程名后使用中文逗号：`Python 程序设计，1,5,6,...`
- 老师和地点合并：`张三，教学楼 101`
- 周次写成：`6-12 双周`
- 周次没有加引号：`5-7,12`

如果 `weeks` 里出现课程代码或字母，例如 `F422200006024`，软件会拒绝这一行，避免导入错误课程。
